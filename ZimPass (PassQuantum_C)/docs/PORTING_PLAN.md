# PassQuantum Go -> ZimPass C Porting Plan

## Source of Truth
- `new-passquantum/GO_APP_SPECIFICATION.md`

## Module Mapping
- Go `core/crypto` -> C `src/crypto`
- Go `core/storage` -> C `src/storage`
- Go `core/model` -> C `src/model`
- Go `core/biometric` -> C `src/biometric`
- Go `ui` -> C `src/ui`
- Go `cmd` and `.tmp_model_src` -> C `apps`

## Compatibility Requirements
- Keep vault file parsing compatible with Go `VaultFile` format.
- Keep app security metadata fields and version semantics compatible.
- Keep KDF parameter encoding and defaults equivalent.
- Keep biometric threshold and feature vector serialization behavior equivalent.

### App Security Metadata Field Compatibility (Detailed)
- `format_version` must map to Go `AppSecurityFormatVersion` semantics.
- `private_key_fingerprint` must preserve byte-for-byte fingerprint behavior.
- `kdf_params` fields (`salt`, `memory`, `iterations`, `parallelism`) must remain equivalent to Go defaults and serialization intent.
- `verifier` must represent deterministic verification material derived from master-password flow.
- `biometric.enabled`, `biometric.threshold`, `biometric.camera_index` must preserve behavior and defaults.
- `biometric_template` must remain binary-compatible as serialized feature-vector bytes.
